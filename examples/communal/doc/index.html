Руководство пользователя

Модуль ЖКХ

Модуль предназначен для создания и управления структурой и иерархией коммунального хозяйства, для связи узлов этой иерархии с объектами PILOT, которые обрабатывают датчики, а также для отображения состояния датчиков, проблемных событий и их привязки к карте.

Основные задачи

  • Создание и поддержка структуры объектов коммунального хозяйства.
  • Привязка узлов структуры к объектам PILOT.
  • Просмотр состояния датчиков по выбранной ветке.
  • Поиск неактивных и проблемных датчиков.
  • Подготовка данных для отображения проблем на карте.

Назначение

  • Для операторов диспетчерских служб.
  • Для инженеров эксплуатации.
  • Для специалистов мониторинга инфраструктуры.
  • Для быстрого обзора состояния объектов.

Как устроен экран

Рабочая область разделена на три части. Они позволяют перейти от структуры объектов к датчикам, а затем к анализу проблем и отображению их на карте.

Левая панель

Структура объектов

Дерево коммунального хозяйства для навигации по районам, улицам, домам и другим объектам.

Центральная панель

Список датчиков

Таблица датчиков выбранного узла и его дочерних элементов с группировкой по объектам.

Правая панель

Сводка и карта

Итоговые показатели и карта для анализа проблемных датчиков и событий.

Основной сценарий работы

1
Выберите узел в дереве
Пользователь выбирает интересующий объект или ветку структуры.
2
Получите связанные объекты PILOT
Модуль собирает привязанные объекты PILOT для выбранного узла и его потомков.
3
Просмотрите статусы датчиков
В таблице отображаются названия датчиков, группы, значения и время последнего обновления.
4
Оцените сводку
Правая панель показывает количество активных, неактивных и проблемных датчиков.
5
Перейдите к анализу на карте
Проблемные датчики могут быть показаны на карте для локализации и последующих действий.

Сводные показатели

Итоговые поля помогают быстро оценить состояние выбранной ветки.

Total Общее количество датчиков.
Active Датчики, обновленные менее часа назад.
Not Active Датчики, которые давно не обновлялись.
Issues Датчики, требующие внимания оператора.

Что показывает таблица датчиков

  • Object — объект, к которому относится датчик.
  • Название — имя датчика.
  • Group — группа или категория датчика.
  • Показатель — текущее значение.
  • Обновление — время последнего обновления.
  • Неполадки — визуальный статус датчика.

Для быстрого поиска используйте строку фильтрации в верхней части таблицы.

Обозначения статусов

Норма — датчик работает корректно.
Внимание — требуется проверка.
Проблема — состояние требует реакции.

Практическое назначение

Модуль помогает работать с коммунальной инфраструктурой как с единой системой. Пользователь может открыть нужный участок структуры, увидеть связанные объекты PILOT, оценить состояние датчиков и подготовить анализ проблемной ситуации на карте.

Это удобно, когда важно контролировать не отдельное устройство, а всю ветку инфраструктуры: например дом, квартал, улицу или район.

Ключевая идея

Единый контекст работы.

Выбор узла в дереве определяет, какие датчики попадут в таблицу, какие показатели будут рассчитаны в summary и какие объекты могут быть показаны на карте.

Оформление страницы упрощено и приближено к спокойному стилю документации PILOT: [doc.pilot-gps.com](https://doc.pilot-gps.com/).
