Ты должен разработать расширение для PILOT по бизнес-идее ниже.

Перед кодом обязательно прочитай репозиторий:

1. AI_SPECS.md
2. docs/AI_EXTENSION_GUIDE.md
3. docs/README_RU.md
4. docs/PILOT_RUNTIME_UTILS_RU.md
5. ближайший пример из examples/

Строгие правила:

- PILOT Extension не является standalone web app.
- Не создавай React/Vue/SPA/Vite/Webpack/npm-проект.
- Не загружай Ext JS вручную.
- Не мокай и не заменяй window.skeleton.
- Extension может использовать объекты и классы, уже загруженные в скомпилированном PILOT app.js.
- Extension должен использовать Store.* для своих бизнес-классов и может использовать доступные host-классы `Pilot.utils.*`, например `Pilot.utils.Toggle`, `Pilot.utils.LeftBarPanel`, `Pilot.utils.ColorField`.
- Можно использовать уже загруженные PILOT библиотеки: Highcharts, Highcharts sunburst, jQuery.
- Для единиц измерения используй window.uom / UOM и helpers speedSS, mileageSS, volumeSS, consSS, amountSS, weigSS, если они доступны.
- Для форматирования данных используй runtime renderers num, dateTimeStr, secondsToHumanTime, makeAddress, makeGeoZone, если они доступны.
- Не подключай повторно Highcharts, jQuery и helper scripts, если они уже доступны в PILOT runtime.
- Если нужны свои CSS-цвета, используй палитру Tailwind CSS для hex-значений, но не подключай Tailwind CSS как framework без отдельной необходимости.
- Runtime-логика должна начинаться только из Module.js.
- Module.js должен содержать Ext.define('Store.<safe_namespace>.Module', { extend: 'Ext.Component', initModule: function () { ... } }).
- doc/index.html обязателен, но это только статическая документация без <script>.
- Если нужна своя вкладка, добавляй Ext component в skeleton.navigation и основную панель в skeleton.mapframe, связывай navTab.map_frame = mainPanel.
- Если нужна команда для объекта Online, добавляй пункт в skeleton.navigation.online.online_tree.contextmenu или context_menu, не заменяя существующее меню.
- Если нужно работать с существующей картой Online, используй getActiveTabMapContainer() или window.mapContainer.
- Если нужно работать с History, используй window.historyMapContainer.
- Если показываешь транспорт/объекты PILOT, загружай данные из /ax/tree.php с params { vehs: 1, state: 1 } и парси иерархические children.

Известный runtime PILOT:

- window.mapContainer - карта Online.
- window.historyMapContainer - карта History.
- window.skeleton.header - главный header.
- window.skeleton.navigation - левая навигация.
- window.skeleton.navigation.online.online_tree - дерево объектов Online.
- window.skeleton.navigation.history.objects - дерево объектов History.
- window.skeleton.navigation.reports.objects - дерево объектов Reports.
- window.skeleton.navigation.reports.reports_store - store отчетов.
- window.skeleton.mapframe - основной контейнер карт/панелей в примерах репозитория.
- window.skeleton.map_frame - то же смысловое место в некоторых описаниях runtime; если сомневаешься, используй fallback skeleton.mapframe || skeleton.map_frame.
- navTab.map_frame - свойство связи компонента навигации с основной панелью.

Результат должен включать:

- структуру файлов;
- полный код Module.js;
- полный код всех дополнительных JS/CSS/backend-файлов, если нужны;
- полный doc/index.html без <script>;
- краткую инструкцию по публикации Module.js;
- список самопроверок по AI_SPECS.md.
